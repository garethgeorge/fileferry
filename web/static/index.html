Appended to the file URL, e.g. /9ef-p9m2rr-my-notes.txt. A trailing extension (my-notes.md) replaces the real file extension for text content.

A fresh AES-256 key is generated for each file and passed to the server to encrypt — never stored on it. The key rides in the share link's #fragment, so it stays out of server logs. Keys are also saved in this browser's local storage (cleared after 48h) so your file list can rebuild working links; lose the link and this browser, and the file can't be decrypted.

Upload from the terminal

Uploading is a two-step API — create a slot, then PUT the bytes. These snippets need jq. If your instance is behind reverse-proxy auth, add your credentials (e.g. curl -u user:pass).
